“好代码”往往不是最炫的那种，而是最能支撑未来变化的那种。

## 放到这次博客改造里

如果一开始就做特别重的技术栈，你很可能会被配置和工程细节拖住。

而当前这套做法是：

- 前台页面优先可读和可收录
- 部署路径优先稳定
- API 能力先预留，不急着全部实现

这比一次性堆满复杂功能更适合个人博客。
